# Documentación para Agentes IA

Esta carpeta contiene documentación específica para agentes de IA que trabajan en el proyecto Raioz.

## 📚 Archivos Disponibles

### `DEFINICIONES_PROYECTO.md`

Documento principal con todas las definiciones, arquitectura, principios de diseño y contexto del proyecto. **Empezar aquí**.

Contiene:

- Visión general del proyecto
- Principios de diseño
- Arquitectura detallada
- Flujos de ejecución
- Formato de configuración
- Decisiones importantes
- Casos de uso

### `CODIGO_STANDARDS.md`

Estándares de código Go y reglas del proyecto.

Contiene:

- Regla de 400 líneas máximo por archivo
- Regla de 120 caracteres máximo por línea
- Estándares de naming
- Buenas prácticas de Go
- Estructura de código
- Ejemplos de código bueno vs malo

### `REGLA_400_LINEAS.md`

Guía detallada sobre cómo dividir archivos que excedan 400 líneas.

Contiene:

- Explicación de la regla
- Excepciones
- Estrategias de división
- Ejemplos prácticos

## 🎯 Uso Recomendado

### Para Agentes IA Nuevos

1. Leer `DEFINICIONES_PROYECTO.md` primero para entender el contexto
2. Revisar `CODIGO_STANDARDS.md` para reglas de código
3. Consultar `REGLA_400_LINEAS.md` si necesitas dividir archivos grandes

### Al Trabajar en Tareas

1. Consultar `DEFINICIONES_PROYECTO.md` para entender el dominio
2. Seguir estándares en `CODIGO_STANDARDS.md`
3. Verificar límites de líneas antes de crear archivos grandes

### Al Revisar Código

1. Verificar que cumple con estándares
2. Confirmar que archivos no exceden 400 líneas
3. Verificar que líneas no exceden 120 caracteres

## 🔄 Mantenimiento

Estos archivos deben actualizarse cuando:

- Se toman decisiones arquitectónicas importantes
- Se cambian estándares de código
- Se agregan nuevos patrones o prácticas
- Se identifican problemas comunes

## 📝 Convenciones

- Todos los archivos en Markdown
- Lenguaje claro y específico
- Ejemplos cuando sea posible
- Referencias a código existente cuando sea útil
- Actualizaciones cuando cambian decisiones
