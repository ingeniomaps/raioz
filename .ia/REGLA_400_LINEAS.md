# Regla: Máximo 400 Líneas por Archivo

## 📏 Regla

**Todos los archivos de código fuente (`.go`) deben tener un máximo de 400 líneas.**

Si un archivo excede este límite, debe dividirse en múltiples archivos más pequeños.

## ✅ Excepciones

1. **Archivos generados automáticamente**

   - Marcados con `// Code generated by...`
   - Estos archivos pueden exceder el límite

2. **Archivos de test**
   - `*_test.go` pueden tener hasta 500 líneas
   - Esto permite tests extensos con múltiples casos

## 🔍 Cómo Verificar

### Manualmente

```bash
wc -l internal/**/*.go
```

### Automáticamente (script)

```bash
find . -name "*.go" ! -name "*_test.go" ! -path "*/vendor/*" -exec sh -c 'lines=$(wc -l < "$1"); if [ "$lines" -gt 400 ]; then echo "$1: $lines lines"; fi' _ {} \;
```

## ✂️ Cómo Dividir un Archivo

### Estrategia 1: Por Responsabilidad

Si un archivo maneja múltiples responsabilidades, dividir por tipo:

```
Antes:
docker.go (450 líneas) - maneja compose y runner

Después:
docker/
├── compose.go    (250 líneas) - generación de compose
└── runner.go     (200 líneas) - ejecución de docker
```

### Estrategia 2: Por Tipo de Función

Separar funciones públicas de privadas/helpers:

```
Antes:
env.go (450 líneas) - todo mezclado

Después:
env/
├── resolver.go   (250 líneas) - funciones públicas
└── helpers.go    (200 líneas) - funciones privadas/helpers
```

### Estrategia 3: Por Dominio

Dividir por conceptos relacionados:

```
Antes:
config.go (500 líneas) - tipos, carga, validación

Después:
config/
├── types.go      (150 líneas) - definiciones de tipos
├── loader.go     (150 líneas) - carga de archivos
└── validator.go  (200 líneas) - validaciones
```

## 📝 Ejemplos

### ✅ Archivo Correcto

```go
// internal/docker/compose.go
// 250 líneas - Genera docker-compose desde configuración

package docker

import (...)
// ... código ...
```

### ❌ Archivo que Necesita División

```go
// internal/docker/all.go
// 450 líneas - Mezcla generación y ejecución

package docker

import (...)
// ... demasiado código mezclado ...
```

### ✅ Archivos Divididos Correctamente

```go
// internal/docker/compose.go
// 280 líneas - Solo generación de compose

// internal/docker/runner.go
// 150 líneas - Solo ejecución de comandos docker
```

## 🎯 Objetivo

El objetivo es mantener archivos:

- **Legibles**: fáciles de entender
- **Enfocados**: una responsabilidad clara
- **Mantenibles**: fáciles de modificar
- **Testeables**: fáciles de testear por separado

## 🔧 Herramientas

### Linter (Futuro)

Se puede crear un linter personalizado o usar reglas de `golangci-lint` para verificar esto automáticamente.

### Pre-commit Hook (Recomendado)

Agregar verificación en pre-commit hook para prevenir commits con archivos demasiado largos.
